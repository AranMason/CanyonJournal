This is a planning doc for adding social features to the Application.

>[Warning] Adding Social Features, is a potential breach of the agreement with Canyon Log, as the objective is to have thier website be the source of truth for recent descents. We don't want to confuse this.


# Potential Feature Set

Feature | Description | Requirements
---|---|---
Unique Readable User Names | Allowing users to have `@name` or similar style unique identifiers | User Settings Page to edit/view identifier; Autogenerate ID; User Table Column
Profile Pictures | Edit and Update Profile Photos | User Settings Page; Auth0 management for Profile Picture?
Profile Page
Social Links | Find people on different platforms
Posts of recent descents
Notifications
Reaction (Kudos, Like etc?)
Friends List
Friends Map | View general region various canyoning connections reside, to be able to organise international trips
Achievements
QR Code for quick adding friends
Flagging 'Records' as Private
Public Comments vs. Private Records 
Linking shared trips together | Could include average Water/Star rating of the trip?
Backlog of Records | Likely just mark as "Private" initally?
Moderation | How do you moderate a social platform?

## Questions
- How to handle posts with Custom Canyons?
- Data Relationship between a Record and a Post
- *[Important]* We do not want to become a secondary source of recent trips. Those should still be the sources like Rope Wiki and Canyon Log - This would be a change in the agreement with CanyonLog
- How do we handle 'Custom' canyons?
